1. Premise & starting point

Leiden clustering on the morphological feature matrix X already produces good results. The task is to integrate the mosaic constraint — specifically, that cells of the same type should not be spatial neighbors — without abandoning the feature signal that Leiden currently exploits well.

The key input is the pairwise IoU matrix between convex hulls of dendritic arbors (already implemented in overlap.py). This matrix is:

  • Sparse in practice — IoU is zero unless cells are in the same spatial neighbourhood.
  • Already available per cell, not per type — so it can be used at graph-construction time, before any clustering decision is made.
  • Type-agnostic — it requires only the cell polygons, no label information.
Cells with high mutual IoU occupy overlapping spatial territories. The mosaic prior says this should be a strong signal that they belong to different types — so high IoU should lower (or remove) the Leiden graph edge between them.

2. Core idea: IoU-weighted repulsion

The standard Leiden / Louvain pipeline builds a k-nearest-neighbour (k-NN) graph in feature space, assigns edge weights by feature similarity (e.g. cosine similarity or RBF kernel on X), then maximises modularity. The natural place to insert the spatial constraint is at edge weight construction:

w(i, j) = similarity(Xᵢ, Xⱼ) × repulsion(IoU(hullᵢ, hullⱼ))

where repulsion(·) is a function that maps IoU to a weight in [0, 1], with:

  • repulsion(0) = 1 — no spatial overlap, feature similarity is unchanged
  • repulsion(high) → 0 — heavy overlap, edge is suppressed regardless of feature similarity

Leiden then sees a graph where spatially overlapping cells are weakly connected, even if featurally similar, and will naturally put them in separate communities.

Alternatively the repulsion can be expressed as an additive penalty subtracted from the affinity, rather than a multiplicative gate. The two are not equivalent: multiplicative means "this edge carries no information" while additive means "this edge is active repulsion".

3. Graph construction variants

V1 — Hard threshold

Remove edges above an IoU cutoff τ

Build the feature k-NN graph as usual. Then, for every edge (i, j) where IoU(i, j) > τ, delete the edge. Leiden runs on the pruned graph.

Pros

  • Simple — only one new parameter (τ)
  • Leaves the feature graph structure fully intact for non-overlapping cells
  • Easy to audit: you can inspect which edges were removed

Cons

  • τ must be set globally — problem if characteristic IoU differs by cell type
  • Hard boundary: IoU=τ−ε keeps edge, IoU=τ+ε drops it — discontinuous
  • Doesn't distinguish "a lot of overlap" from "a little overlap above threshold"
V2 — Soft multiplicative

Downweight edges continuously with IoU

Multiply the feature-based edge weight by a smooth repulsion function. A sigmoid works well here:

repulsion(u) = 1 − σ((u − τ) / s) where σ is the logistic function, τ is the inflection point, s controls steepness.

At u≪τ, the edge weight is barely changed. At u≫τ, the edge weight collapses to zero. The soft version makes the optimisation landscape smoother and is more robust to noisy hull estimates.

Pros

  • Continuous — avoids cliff effects at the threshold
  • Two parameters (τ, s) give independent control over "where repulsion kicks in" and "how sharp it is"
  • Naturally handles measurement noise in hull estimates

Cons

  • Still requires choosing τ — the non-uniform threshold problem applies here too
  • Very similar to V1 in practice if s is small
V3 — Separate repulsion graph

Combine two graphs: feature affinity + spatial repulsion

Build two graphs independently: (1) the usual feature k-NN graph with positive weights, (2) a spatial repulsion graph where edges connect cells with IoU > τ_low with weight proportional to their excess IoU. Subtract (2) from (1) to get the combined adjacency matrix, then run Leiden with a signed-modularity objective (RB Potts model or CPM support for leidenalg).

Pros

  • Explicit separation of "attraction" (feature) and "repulsion" (spatial)
  • Can tune the magnitude of each independently via a mixing parameter λ
  • The repulsion graph is sparse and can be computed only for spatial neighbours

Cons

  • Negative edge weights complicate the modularity objective — not all Leiden implementations handle them
  • One additional hyperparameter (λ) on top of τ
  • Subtraction can produce a disconnected graph if repulsion is too strong
V4 — Graph rewiring

Replace suppressed edges with long-range feature connections

When an edge is suppressed by high IoU, don't just delete it — rewire it to the next-best feature-similar cell that does NOT have high IoU with cell i. This preserves the graph degree distribution (important for Leiden's resolution stability) while enforcing the spatial constraint.

Pros

  • Each cell retains the same number of edges in the graph
  • Actively promotes connections to spatially non-overlapping but featurally similar cells — these are precisely the same-type cells at mosaic spacing
  • Resolution / K is less perturbed compared to simple edge deletion

Cons

  • More complex to implement
  • If no non-overlapping similar cell exists within a reasonable distance, the replacement is a worse feature match — may introduce noise
  • Adds another parameter: how far to search for the replacement

4. The BC minimal-overlap insight

Biological observation (BCs) For bipolar cells, the tips of neighboring cells of the same type barely touch, resulting in ~10% IoU between adjacent same-type pairs. This is not overlap in the pathological sense — it is a structural feature of how BCs tile space. It means:
  • IoU ≈ 0% between two cells → they are well-separated; no information about type
  • IoU ≈ 10% → characteristic of same-type spatial neighbors (BC case)
  • IoU > 30–40%? → substantial territorial overlap → likely different types sharing a niche

This breaks the naive assumption that "any IoU > 0 is repulsion". The repulsion function should be non-monotone in the biologically relevant range — or at minimum, the repulsion threshold τ must sit above the characteristic same-type contact level.

A plausible functional form that accommodates this:

repulsion(u) = max(0, (u − τ_low) / (τ_high − τ_low)) for τ_low < u < τ_high = 1 for u ≥ τ_high where τ_low ≈ characteristic same-type contact IoU (~10% for BCs) τ_high = IoU level where territorial overlap is clearly pathological (~40%?)

Below τ_low, no repulsion is applied — touching cells might well be same-type neighbors. Above τ_high, full repulsion. In between, a ramp. The key unknown is τ_low per cell type, which motivates the iterative approach in Section 6.

Complication: is 10% IoU universal for same-type BCs, or does it depend on cell size / local density? If the characteristic contact IoU varies spatially (e.g. denser patches → more touching) or by subtype, a single τ_low for all BCs may still be wrong. The right fix might be to normalize IoU by the expected contact area given the local cell spacing — but this requires knowing local density, which is type-dependent.

5. The non-uniform threshold problem

The core difficulty: different cell types have different coverage scales. A large WFAC and a small NFAC sitting next to each other will have a very different IoU than two WFACs of the same type, or two NFACs. The same IoU value means something different depending on the sizes involved.

Three rough ways to think about this:

5a. Raw IoU is scale-invariant — but that may be misleading

IoU = intersection / union, which is already area-normalized. Two cells of very different sizes with 20% IoU is a different situation from two same-size cells with 20% IoU: the smaller cell's territory is substantially occupied by the larger one. One alternative metric: overlap coefficient = intersection / min(area₁, area₂), which measures "how much of the smaller cell is covered by the larger". This may be more biologically meaningful than raw IoU for mixed-size pairs.

5b. Cell-size-specific thresholds

Bin cells by hull area (or dendritic field diameter) into size classes. Set τ independently per size class, since characteristic touching IoU will be similar within a size class. This is essentially a coarser version of the iterative approach — precomputed from rough size estimates rather than learned from clustering iterations.

5c. Local relative threshold

For each cell i, compute the distribution of IoU values with all cells within a radius R. The repulsion threshold for cell i is set at a percentile of this local distribution — e.g. τᵢ = 75th percentile of IoU(i, neighbours). This adapts per cell without requiring knowledge of cell type, but it's susceptible to being dominated by a few large nearby cells.

6. Iterative threshold refinement

The most principled approach, at the cost of complexity: use the clustering output itself to calibrate the threshold, then re-run.

Initialize: feature-only Leiden

Run standard Leiden on the feature k-NN graph. This gives an initial (biased) clustering with no spatial constraint. Use it only to bootstrap threshold estimates — don't trust the assignments themselves yet.

Estimate per-cluster IoU baseline

For each cluster C, find pairs of cells (i, j) where both i, j ∈ C and they are spatial neighbours (e.g. IoU > 0 or within distance d). The distribution of these within-cluster IoU values gives the characteristic same-type contact IoU for cluster C. Let τ_low(C) = median or 75th percentile of this distribution.

Assign per-cell τ_low from cluster membership

For each cell i, set τ_low(i) = τ_low(cluster(i)). This is now a per-cell repulsion onset that reflects the type-specific contact IoU. Cells not yet in stable clusters can use a global fallback τ_low.

Re-build graph with adaptive repulsion

For edge (i, j): τ_low(i, j) = (τ_low(i) + τ_low(j)) / 2. Apply the repulsion function using this pairwise τ_low. Re-run Leiden.

Iterate until cluster membership stabilizes

Repeat steps 2–4. Convergence criterion: fraction of cells changing cluster assignment falls below ε (e.g. 1%). In practice 2–3 iterations may be sufficient since τ_low estimates stabilize quickly once the major clusters are correct.

Chicken-and-egg problem at step 2 The per-cluster τ_low estimate depends on the current (imperfect) cluster assignment. If the initial Leiden run merges two types that have different contact IoU levels, the estimated τ_low for that merged cluster will be wrong — possibly too high, failing to separate them. The first iteration is therefore the most fragile. One mitigation: use a conservative (low) global τ_low for the first iteration to lean toward over-splitting rather than under-splitting.
What if a cluster contains few spatial neighbours? Small or sparse clusters (e.g. WFACs in a small patch) may have zero or very few within-cluster IoU measurements to estimate τ_low from. The fallback should be either the global median τ_low or the τ_low inherited from the nearest same-size cluster. Alternatively: only apply the spatial constraint to cells that have a minimum number of spatial neighbours.
Potential shortcut: hull-area-normalized τ_low Instead of iterating, estimate τ_low directly from cell geometry. For a cell with hull area A, the expected IoU with a touching same-type neighbour of similar size is approximately proportional to the contact perimeter × a characteristic overlap depth, divided by 2A. For regularly-spaced cells: τ_low ≈ f(A, local_density). This avoids iteration entirely but requires a geometric model of the expected contact — probably a reasonable approximation for BCs and RGCs, less so for ACs.

7. Implementation sketch

Concretely, the approach maps onto the existing codebase as follows:

Inputs

  • dataset.X — morphological features, shape (n_cells, n_dims)
  • dataset.centers — cell positions, shape (n_cells, 2) — for spatial neighbourhood query
  • dataset.polygons — convex hulls per cell — for IoU via overlap.compute_iou_matrix()

Graph construction (pseudo-Python)

from sklearn.neighbors import kneighbors_graph import igraph, leidenalg # 1. Feature k-NN graph A_feat = kneighbors_graph(dataset.X, n_neighbors=k, mode='distance', ...) # convert distances to similarities, e.g. RBF kernel # 2. Sparse IoU matrix — only compute for spatially close pairs spatial_pairs = get_pairs_within_radius(dataset.centers, r=mosaic_radius) iou_vals = {(i,j): compute_hull_iou(hull_i, hull_j) for i,j in spatial_pairs} # 3. Build combined weight matrix for (i, j), iou in iou_vals.items(): tau_low = tau_low_per_cell[i] # scalar, ~0.1 for BCs tau_high = tau_high # global, ~0.4 repulsion = max(0, min(1, (iou - tau_low) / (tau_high - tau_low))) A_feat[i, j] *= (1 - repulsion) A_feat[j, i] *= (1 - repulsion) # 4. Run Leiden g = igraph.Graph.Weighted_Adjacency(A_feat.toarray(), mode='undirected') partition = leidenalg.find_partition(g, leidenalg.RBConfigurationVertexPartition, weights='weight', resolution_parameter=res)

Key parameters to expose

  • k — number of feature neighbours in the base graph
  • r — spatial radius for IoU computation (set to ~1.5× expected mosaic spacing)
  • tau_low — IoU below which no repulsion applies (initialize at e.g. 0.05 globally, then update iteratively)
  • tau_high — IoU at which full repulsion is applied
  • res — Leiden resolution parameter (controls K)
  • n_iter — number of τ_low update iterations

8. Open questions

IoU between Voronoi tiles vs. actual dendritic arbors. The toy simulation uses Voronoi polygons; real data uses convex hulls of reconstructed dendrites. The ~10% touching overlap for BCs is a statement about actual arbors — is it reproduced by the Voronoi / convex-hull approximation? If the simulation systematically underestimates this contact (because Voronoi tiles don't "touch" by construction), τ_low estimates from simulated data won't transfer to real data.
What is the right alternative metric to raw IoU? For mixed-size cell pairs (WFAC touching NFAC), overlap coefficient (intersection / min_area) may be more meaningful than IoU. Should we switch to overlap coefficient, or use IoU but with size-stratified thresholds? Or provide both and let the downstream user pick?
How many iterations does the adaptive τ_low scheme need in practice? On the toy data we can measure this directly: run iterative refinement and track how much τ_low values change across iterations. If they stabilize in 2–3 rounds, the scheme is viable. If they don't converge, the chicken-and-egg problem is too severe and we need a non-iterative τ_low estimate.
How should repulsion interact with clipped cells? Clipped cells have truncated hulls → their IoU with neighbours is systematically underestimated → repulsion underestimated. Should clipped cells be excluded from the spatial repulsion term entirely, or should their IoU be upscaled by a factor based on the estimated clipping fraction? The latter requires knowing what fraction of the hull is visible.
What happens to resolution / K stability? Standard Leiden resolves K via the resolution parameter. Inserting repulsion effectively removes edges, which changes the effective resolution even at fixed res. Does the τ-repulsion effectively shift K upward (more communities)? If so, we need to re-tune res for each (τ_low, τ_high) pair, which complicates the parameter sweep.
V4 (rewiring) vs. V2 (soft downweight): which is better for K stability? Rewiring preserves node degree and should therefore disturb the resolution–K relationship less. Is this worth the implementation complexity? Easy to test: compare community-size distributions and NMI against ground truth for both on the toy dataset.
How to validate that the spatial constraint is actually helping on toy data? Ground truth labels are available. The metric should be NMI (or ARI) as a function of (τ_low, τ_high, n_iter). But we also want to check that the improvement comes from the spatial constraint and not just from having changed the effective k or resolution — so a null test with random permutation of IoU values would be informative.
Can we estimate τ_low from cell size alone, without iterating? For cells with approximately circular territories of radius ρ, the touching-neighbor contact IoU scales as IoU ≈ (2ρ·δ − δ²·π/4) / (2πρ² − ...) where δ is the overlap depth. If we model δ as a fixed fraction of ρ, τ_low ≈ constant regardless of size — which would be good news. Worth deriving or checking empirically on the simulation.
