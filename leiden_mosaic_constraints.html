1. Premise & starting point

Leiden clustering on the morphological feature matrix X already produces good results. The task is to integrate the mosaic constraint — specifically, that cells of the same type should not be spatial neighbors — without abandoning the feature signal that Leiden currently exploits well.

The key input is a pairwise overlap matrix between convex hulls of dendritic arbors (currently implemented as IoU in overlap.py, see Section 2 for the preferred metric). This matrix is:

  • Sparse in practice — overlap is zero unless cells are in the same spatial neighbourhood.
  • Already available per cell, not per type — so it can be used at graph-construction time, before any clustering decision is made.
  • Type-agnostic — it requires only the cell polygons, no label information.
Cells with high mutual overlap occupy the same spatial territory. The mosaic prior says this is a strong signal that they belong to different types — so high overlap should lower (or remove) the Leiden graph edge between them.
Biological observation: same-type cells can have non-zero overlap (BC example) For bipolar cells, the tips of neighboring cells of the same type barely touch, resulting in a small but non-zero coverage fraction between adjacent same-type pairs (~10% is a rough estimate from convex hull measurements of axonal arbors; the key point is simply that it is > 0). This is not pathological overlap — it is a structural feature of how BCs tile space. Consequently:
  • C ≈ 0 between two cells → well-separated; no information about type membership
  • C ≈ 10% → characteristic of same-type spatial neighbours in BCs — not a repulsion signal
  • C > some threshold τ_high → substantial territorial overlap → likely different types sharing a niche → strong repulsion
This means the repulsion function must have a dead zone at low overlap: repulsion only kicks in above a type-specific threshold τ_low that sits at or above the characteristic same-type contact level. Setting τ_low = 0 would mistakenly penalise same-type BC neighbours.

2. Core idea: coverage-fraction-weighted repulsion

2a. The overlap metric: coverage fraction

Raw IoU (intersection / union) has an undesirable property for mixed-size cell pairs: if a small cell is completely contained within a large one, IoU ≈ area_small / area_large, which can be very small even though the smaller cell has zero independent territory. The biologically relevant question is how much of each cell's territory is shared — which is better captured by:

C(i, j) = intersection(hull_i, hull_j) / min(area_i, area_j)

This is sometimes called the overlap coefficient or containment ratio. Key properties:

  • C = 1 when the smaller cell is fully contained in the larger — maximum repulsion regardless of size ratio.
  • C ≈ IoU when cells are roughly the same size (the common case within a cell type).
  • C is less sensitive to clipping artefacts than IoU, because a truncated hull reduces both intersection and min-area proportionally.

Note: using min rather than max in the denominator is what achieves high repulsion under containment. With max, a fully-contained small cell would give C = area_small / area_large ≈ 0 for very different sizes, defeating the purpose.

2b. The repulsion function

The standard Leiden pipeline builds a k-NN graph in feature space, weights edges by feature similarity (e.g. RBF kernel on X), then maximises modularity. The spatial constraint enters at edge weight construction:

w(i, j) = similarity(Xᵢ, Xⱼ) × repulsion(C(hullᵢ, hullⱼ))

The repulsion function incorporates the dead zone established in Section 1:

repulsion(c) = 0 for c ≤ τ_low (dead zone) = (c − τ_low) / (τ_high − τ_low) for τ_low < c < τ_high (ramp) = 1 for c ≥ τ_high (full suppression) where τ_low = characteristic same-type contact C (e.g. ~0.10 for BCs; type-specific) τ_high = C level where territorial overlap is clearly pathological (e.g. ~0.40)

Leiden then sees a graph where cells with substantial overlap are weakly connected even if featurally similar, and will naturally place them in separate communities.

τ_high is also type-dependent and unknown a priori. For BCs where same-type contact C ≈ 10%, a reasonable τ_high might be ~40%. But for RGCs with smaller territories or NFACs with denser packing, the "pathological" overlap level may differ. In practice τ_high is probably less sensitive than τ_low — a global τ_high ≈ 0.4 may be acceptable as a starting point, while τ_low needs per-type calibration (see Section 5).

3. Graph construction variants

V1 — Hard threshold

Remove edges above a coverage fraction cutoff τ_high

Build the feature k-NN graph as usual. For every edge (i, j) where C(i, j) > τ_high, delete the edge. The dead zone (C < τ_low) is implicit: C below τ_low is simply not large enough to trigger deletion. Leiden runs on the pruned graph.

Pros

  • Simple — adds only τ_low and τ_high as new parameters
  • Leaves the feature graph fully intact for non-overlapping cells
  • Easy to audit: inspectable which edges were removed and why

Cons

  • Hard boundary at τ_high — discontinuous response to overlap
  • No gradation: C just above τ_high is treated the same as C = 1
  • τ_low must still be chosen to define the dead zone
V2 — Soft multiplicative (recommended starting point)

Downweight edges continuously via the ramp function

Use the piecewise-linear ramp from Section 2b directly as the multiplicative weight modifier. The dead zone, ramp, and saturation are all encoded in a single smooth function controlled by (τ_low, τ_high). No additional parameters.

Pros

  • Directly implements the biologically motivated dead zone + ramp
  • Continuous — avoids cliff effects; robust to noise in hull estimates
  • Minimal parameters: τ_low (per-type, see Section 5) and a single global τ_high
  • Easy to initialise: start with global τ_low, refine iteratively

Cons

  • The ramp shape is fixed (linear) — a sigmoid might be more appropriate if the transition is sharp
  • Multiplicative suppression means the edge carries no information, not active repulsion — may under-enforce in very dense overlapping regions
V3 — Separate repulsion graph

Combine feature affinity graph + explicit spatial repulsion graph

Build two graphs independently: (1) the usual feature k-NN graph with positive weights, (2) a spatial repulsion graph where edges connect cells with C > τ_low with weight proportional to their excess coverage fraction. Subtract (2) from (1), scaled by a mixing parameter λ, then run Leiden with a signed-modularity objective (RB Potts or CPM via leidenalg).

Pros

  • Explicit separation of attraction (feature) and repulsion (spatial)
  • λ gives independent control over repulsion magnitude
  • Repulsion graph is sparse — only computed for spatial neighbours

Cons

  • Negative edge weights are non-standard — not all Leiden backends handle them
  • One additional hyperparameter (λ) on top of τ_low and τ_high
  • Subtraction can produce a disconnected graph if repulsion is too strong
V4 — Graph rewiring

Replace suppressed edges with long-range feature connections

When an edge is suppressed by high C(i, j), don't delete it — rewire it to the next-best feature-similar cell that does NOT have high coverage fraction with cell i. This preserves the graph degree distribution (important for Leiden's resolution stability) while enforcing the spatial constraint. The rewired edges actively promote connections to spatially non-overlapping but featurally similar cells — precisely the same-type cells at mosaic spacing.

Pros

  • Degree distribution is preserved → K / resolution relationship is less perturbed
  • Actively promotes cross-mosaic-spacing connections rather than just deleting local ones

Cons

  • More complex to implement
  • If no non-overlapping similar cell exists nearby, the replacement is a worse feature match
  • Adds a search-radius parameter for the replacement candidate

4. The non-uniform threshold problem

Different cell types have different coverage scales. The characteristic same-type contact level (τ_low) cannot be set uniformly across all types — a value appropriate for BCs may be far too high or too low for RGCs or ACs. The coverage fraction metric (Section 2a) already mitigates the size-mismatch problem for τ_high: a small cell that is fully contained in a large one gets C = 1 regardless of the size ratio. But τ_low still requires per-type calibration.

4a. Cell-size-specific initial estimate

Bin cells by hull area into size classes. For cells of similar size, IoU and coverage fraction are approximately equal, and the characteristic contact C is roughly constant within a size class. Use a precomputed or literature-derived τ_low per size class as the starting point before iterative refinement. For cell types where mosaic contact is not expected (WFACs, geometrically unusual cells), skip the spatial constraint entirely.

4b. Local relative estimate

For each cell i, compute the distribution of C values with all cells within a spatial radius R. Set τ_low(i) as a percentile (e.g. 25th) of this distribution — the idea being that the lowest non-zero C values in the neighbourhood reflect typical same-type touching, while outliers above reflect genuine overlap. This adapts per-cell without requiring any type knowledge, but can be noisy in sparse regions.

5. Iterative threshold refinement

The most principled approach: use the clustering output itself to calibrate τ_low, then re-run. Start with a single global τ_low (conservative — lean toward over-splitting to avoid the chicken-and-egg failure mode described below), then refine per cluster across 2–3 iterations. Abort and revert to the global τ_low if cluster assignments do not stabilise.

Initialize: Leiden with global τ_low

Run Leiden with the repulsion function using a single conservative global τ_low (e.g. 0.0 or a small value like 0.05). This initial run is biased — it over-splits where same-type contact C exceeds τ_low — but it provides cluster assignments needed to bootstrap per-cluster estimates.

Estimate per-cluster C baseline

For each cluster C_k, find all within-cluster spatial neighbour pairs (i, j) with i, j ∈ C_k and C(i, j) > 0. The distribution of these within-cluster coverage fractions gives the characteristic same-type contact for cluster C_k. Set τ_low(C_k) = median (or a low percentile such as 25th) of this distribution.

Assign per-cell τ_low from cluster membership

For each cell i, τ_low(i) = τ_low(cluster(i)). Cells in small clusters with too few within-cluster spatial neighbours to estimate τ_low reliably fall back to the global τ_low.

Rebuild graph with per-cell adaptive repulsion

For edge (i, j): use τ_low(i, j) = (τ_low(i) + τ_low(j)) / 2 as the pairwise dead-zone onset. Apply the ramp repulsion function with this adaptive τ_low and the global τ_high. Re-run Leiden.

Iterate; stop when stable or after 2–3 rounds

Repeat steps 2–4. Convergence criterion: fraction of cells changing cluster assignment falls below ε (e.g. 1%). If assignments have not stabilised after 3 iterations, abort the adaptive scheme and return the result from the global-τ_low run — this indicates that the chicken-and-egg instability is too severe for the current dataset.

Chicken-and-egg problem at step 2 The per-cluster τ_low estimate depends on the current (imperfect) assignment. If the initial run merges two types whose contact C levels differ, the estimated τ_low for the merged cluster will be wrong — possibly too high, preventing separation on the next iteration. Starting with a conservative (low) global τ_low leans toward over-splitting, which is the safer failure mode: over-split clusters can be merged post-hoc, while under-split clusters are harder to detect.
What if a cluster contains few spatial neighbours? Small or sparse clusters (e.g. WFACs with very few representatives in the patch, or cells with mostly clipped hulls) may have too few within-cluster C measurements to form a reliable distribution. Fall back to the global τ_low for these cells, and consider excluding them from the spatial constraint entirely if fewer than a minimum number of spatial neighbours exist.
Potential non-iterative shortcut For cells with approximately circular territories of uniform size within a type, the expected same-type contact C is roughly constant and depends only on the characteristic overlap depth relative to cell radius. This means τ_low could be estimated directly from cell hull area and local spatial density — avoiding iteration entirely. Probably a reasonable approximation for BCs and RGCs; less so for ACs with irregular geometries.

6. Implementation sketch

Concretely, the approach maps onto the existing codebase as follows:

Inputs

  • dataset.X — morphological features, shape (n_cells, n_dims)
  • dataset.centers — cell positions, shape (n_cells, 2) — for spatial neighbourhood query
  • dataset.polygons — convex hulls per cell — for coverage fraction computation

Coverage fraction (replaces raw IoU in overlap.py)

def coverage_fraction(hull1, hull2): poly1, poly2 = Polygon(hull1), Polygon(hull2) intersection = poly1.intersection(poly2).area min_area = min(poly1.area, poly2.area) return intersection / min_area if min_area > 0 else 0.0

Graph construction (pseudo-Python)

from sklearn.neighbors import kneighbors_graph import igraph, leidenalg # 1. Feature k-NN graph A_feat = kneighbors_graph(dataset.X, n_neighbors=k, mode='distance', ...) # convert distances to similarities, e.g. exp(-d^2 / sigma^2) # 2. Sparse coverage-fraction matrix — only for spatially close pairs spatial_pairs = get_pairs_within_radius(dataset.centers, r=mosaic_radius) cf_vals = {(i,j): coverage_fraction(hull_i, hull_j) for i,j in spatial_pairs} # 3. Build combined weight matrix using per-cell adaptive τ_low for (i, j), cf in cf_vals.items(): tau_low_ij = (tau_low_per_cell[i] + tau_low_per_cell[j]) / 2 repulsion = max(0.0, min(1.0, (cf - tau_low_ij) / (tau_high - tau_low_ij))) A_feat[i, j] *= (1.0 - repulsion) A_feat[j, i] *= (1.0 - repulsion) # 4. Run Leiden g = igraph.Graph.Weighted_Adjacency(A_feat.toarray(), mode='undirected') partition = leidenalg.find_partition(g, leidenalg.RBConfigurationVertexPartition, weights='weight', resolution_parameter=res)

Key parameters to expose

  • k — number of feature neighbours in the base graph
  • r — spatial radius for coverage fraction computation (~1.5× expected mosaic spacing)
  • tau_low_global — initial global dead-zone onset (e.g. 0.05); updated iteratively per cluster
  • tau_high — coverage fraction at which full repulsion is applied (e.g. 0.40; global)
  • res — Leiden resolution parameter (controls K)
  • n_iter — max iterations for τ_low refinement (suggest 3; abort if not converged)

7. Mosaic-completeness-guided merging

7a. The core observation

Leiden with spatial repulsion tends to over-split: the resolution parameter that prevents false merges also fragments genuine types into two or more sub-clusters (3+ fragments of the same type are entirely possible). The key insight is that over-split fragments have spatially complementary patterns: if type T got split into fragments A, B, C…, each fragment is only a partial mosaic — irregular spacing, double (or triple) the expected NND — but together they form T's complete, regular mosaic.

This gives a principled post-processing criterion: merge clusters whose union forms a more regular mosaic than they do separately, provided they are also featurally similar. The two conditions are necessary together — spatial complementarity alone would incorrectly merge genuinely distinct types that happen to tile space next to each other (which is actually what a healthy mosaic looks like between types), and feature similarity alone would merge types with overlapping morphology that happen to be spatial neighbours.

Contrast with the repulsion step Leiden repulsion discourages putting spatially overlapping cells together (cells within < 1 cell-diameter). The merge step does the opposite: it identifies pairs that are spatially interleaved at mosaic spacing (one inter-type distance apart) and featurally similar. The two signals operate at different scales and are complementary.

7b. Regularity metric: NND-CV (plug-in design)

The coefficient of variation of within-cluster nearest-neighbour distances (NND-CV) is the primary regularity metric. NND-CV = std(NND) / mean(NND) across all cells in a cluster, where each cell's NND is its distance to the nearest other cell in the same cluster. Interpretation:

  • NND-CV ≈ 0 → perfect grid
  • NND-CV ≈ 0.52 → Poisson (random) point process
  • Real retinal mosaics: typically 0.05–0.25

For a candidate merge of clusters A and B, the regularity gain is:

ΔR(A, B) = NND-CV(A) + NND-CV(B) − 2 × NND-CV(A ∪ B) (positive → merge improves regularity; negative → merge makes things worse)

The metric is intentionally behind a thin interface so it can be swapped for Ripley's K, Voronoi-area CV, or a permutation-test-based score without changing the rest of the algorithm.

NND-CV is noisy for small clusters. With <~20 cells per cluster, NND-CV has high variance. Mitigations: (a) set a minimum cluster size below which merging is not attempted and the cluster is flagged as "too small to evaluate"; (b) add a bootstrap confidence interval around ΔR and only merge when the lower bound is still positive; (c) swap in a more robust metric (Ripley's K) when cluster size falls below the threshold.

7c. PAGA as the feature-similarity gate

PAGA (Partition-based Graph Abstraction, Wolf et al. 2019) computes a cluster-level graph from the existing cell-cell feature k-NN graph. For each pair of clusters (A, B), the PAGA edge weight measures the fraction of possible cross-cluster edges that actually exist, normalised against a null model (random assignment of cells to clusters of the same sizes). A weight close to 0 means the two clusters are no more connected in feature space than expected by chance; a weight close to 1 means they are strongly connected — likely fragments of the same type.

PAGA is implemented in scanpy (already a dependency) and operates directly on the feature k-NN graph built during the Leiden step — no extra graph construction needed.

Merge candidate (A, B) iff: PAGA(A, B) ≥ θ_paga (feature similarity gate) AND ΔR(A, B) ≥ δ (mosaic regularity gate)

Why PAGA rather than a simpler cluster-centroid distance? PAGA accounts for the full distribution of each cluster's cells in feature space, not just the mean — two clusters with similar centroids but non-overlapping distributions will have a low PAGA weight and will correctly not be merged. It also naturally handles the case where a type's feature distribution is non-Gaussian or elongated.

Why not PAGA alone? Two genuine but featurally similar types that tile space together (like two BC subtypes with similar stratification) will have a high PAGA weight because their cells are morphologically close and hence k-NN connected. Without the ΔR gate, PAGA would suggest merging them. The mosaic regularity criterion blocks this: merging two distinct types that already form separate complete mosaics gives a worse joint NND-CV (the merged set is a superposition of two regular grids, which looks irregular at their shared spacing). Only merging genuine fragments improves regularity.

7d. Algorithm: iterative hierarchical merge

The algorithm is iterative rather than a single-pass greedy scan, which naturally handles 3+ fragments: after merging A and B into AB, the PAGA and ΔR scores for (AB, C) are recomputed and may now cross the threshold.

Over-split Leiden + repulsion

Run the Leiden + repulsion algorithm at a resolution that deliberately over-splits (err on the side of too many clusters). This is the safe starting point for a merge-down approach.

Compute PAGA on the current partition

Run scanpy.tl.paga on the feature k-NN graph with the current cluster labels. This gives a K×K matrix of inter-cluster connectivity weights at essentially zero marginal cost (the feature graph already exists).

Filter to PAGA-connected candidate pairs

Retain only pairs (A, B) with PAGA(A, B) ≥ θ_paga. This reduces the O(K²) search space to a sparse set of candidates. For large K (many over-split clusters), most pairs will have near-zero PAGA weight and are immediately eliminated.

Score each candidate by ΔR

For each surviving pair, compute ΔR(A, B) = NND-CV(A) + NND-CV(B) − 2 × NND-CV(A ∪ B). This is the only expensive step: O(N log N) per pair (one KD-tree query per cluster for the within-cluster NND). Skip pairs where either cluster is below the minimum size for reliable NND-CV.

Merge the best-scoring pair; update PAGA and ΔR

Select the pair (A*, B*) with the highest ΔR that also satisfies ΔR ≥ δ. Merge into a single cluster. Recompute PAGA only for the rows/columns of the new cluster (local update — the rest of the PAGA matrix is unchanged). Recompute ΔR only for pairs involving the new cluster.

Repeat until no qualifying pair remains

Stop when no pair satisfies both PAGA(A, B) ≥ θ_paga and ΔR(A, B) ≥ δ simultaneously. Because PAGA is recomputed after each merge, 3+ fragment chains are handled naturally: A merges with B, then AB's updated PAGA detects connectivity with C, then AB merges with C.

Re-run Leiden with merged labels as warm start

Pass the final merged assignments as initial_membership to leidenalg.find_partition. Leiden is free to refine boundaries locally (cells near the boundary of two now-correctly-merged clusters may benefit from reassignment), but the coarse partition is already correct. Use the same resolution as the initial over-split run — the merge step has corrected K without touching the resolution parameter.

PAGA threshold θ_paga: what value? PAGA weights are in [0, 1] and depend on cluster size and graph density. A threshold of ~0.3 is commonly used in single-cell literature, but this may not transfer directly. In practice, the threshold should be validated on the toy dataset by checking what PAGA weight is observed between ground-truth same-type fragments vs. distinct-type pairs. We expect a clear gap.
PAGA recomputation after merge. Full PAGA recomputation after every merge is O(K × N). For moderate K (10–50 clusters) this is cheap. For large K a local update is possible: only the rows and columns of the merged cluster change. The scanpy PAGA implementation may not support partial updates, so we may need to either rerun it fully or implement a lightweight version that works on the sparse adjacency matrix directly.

7e. Mosaic completeness as a secondary signal

For well-characterised types where the expected cell count per patch is known (from literature or from completed proofreading of a reference type), a simple completeness check can flag under-populated clusters as merge candidates even before the PAGA/ΔR test. A cluster with substantially fewer cells than expected (e.g. <60% of the predicted count) is a likely fragment. However, since proofreading is ongoing and completeness is not available for all types, this signal should be advisory only — a soft prior that raises the priority of a candidate pair rather than triggering an unconditional merge.

8. Path transitivity: why edge reduction isn't enough

8a. The core problem

Both the coverage-fraction repulsion (Section 2) and the proximity-based variant tried so far reduce or remove direct edges between cells that are spatial neighbours. This turned out not to be sufficient. The reason is a fundamental property of community detection on graphs:

Community membership propagates through multi-hop paths. Weakening or removing the direct edge (i, j) does not prevent i and j from being in the same community if a strong indirect path i → C → D → j exists in the feature graph.

Concretely, for the anisotropic case: six cells on the feature-space boundary between types 0 and 2 are misassigned to type 2. Each of these cells has many strong feature edges to other type-2 cells (which may be far away spatially). Weakening the spatial edges to their nearby type-2 neighbours does almost nothing — the community cohesion is maintained through the non-spatial feature edges, which are untouched.

The toy experiment confirms this. The proximity-based strategy applies repulsion to 4× as many pairs as the coverage-fraction strategy (different radius). The ARI is unchanged. Adaptive τ_low shifts upward to compensate, neutralising the extra pairs. The result is structurally the same local minimum: feature-graph connectivity dominates spatial repulsion.

8b. Why zero-weight edges are not repulsive

The current multiplicative scheme drives edge weights toward zero for high-repulsion pairs:

W'(i, j) = W_feat(i, j) × (1 − repulsion) ∈ [0, W_feat(i, j)]

A weight of zero is neutral in the Leiden / modularity objective — it neither attracts nor repels. The objective for community assignment δ(c_i = c_j) contains W'(i,j) as a reward for putting i and j together. With W'(i,j) = 0 there is no reward, but also no penalty. Leiden is free to place i and j in the same community if indirect paths make that the better option.

8c. The fix: subtractive (negative-weight) repulsion

To actively penalise putting spatial neighbours together, the edge weight must be pushed below zero:

W'(i, j) = W_feat(i, j) − λ × ramp(signal(i,j), τ_low, τ_high) where signal = coverage fraction C(i,j) (or proximity score 1 − d/r) λ = repulsion magnitude (new hyperparameter, scale = W_feat units) ramp = same piecewise-linear function as before, ∈ [0, 1]

When λ × ramp > W_feat(i,j), the resulting weight is negative. In the RBConfiguration objective used by leidenalg:

H = Σ_{ij} [W'(i,j) − γ · k_i · k_j / (2m)] · δ(c_i = c_j)

a negative W'(i,j) means having i and j in the same community subtracts from H. This is a genuine repulsion term — the community is penalised for containing these two cells together, not merely deprived of a reward. Indirect paths still contribute positively, but with a large enough λ the direct penalty dominates and Leiden is forced to separate i and j.

leidenalg supports negative edge weights in RBConfigurationVertexPartition. The partition objective is defined over the weight matrix directly; negative entries are permitted. The null-model term γ · k_i · k_j / (2m) uses k_i = Σ_j |W(i,j)| (sum of absolute weights) for the degree, so the null model remains well-defined even with negative weights. This needs to be verified empirically, but the mathematical formulation is sound.

8d. The dead zone still applies — and matters more

With subtractive repulsion, the dead zone τ_low is critical. Without it, same-type cells that naturally touch (like BC neighbours with C ≈ 10%) would receive a negative edge weight and be pushed apart — incorrectly breaking the mosaic. The ramp(signal, τ_low, τ_high) function ensures repulsion is zero below τ_low, so the negative-weight regime only activates for pairs well above the characteristic same-type contact level.

The iterative τ_low estimation (Section 5) therefore becomes essential when using subtractive repulsion: if τ_low is too low, genuine same-type pairs get negative weights and are incorrectly separated; if too high, the repulsion never activates. The adaptive estimate — 75th percentile of within-cluster signal values — serves as a natural calibration.

8e. Connection to existing variants and what changes in practice

V3 — Signed graph (revisited)

Subtractive repulsion = V3 with λ chosen per-pair

Section 3's V3 described a separate repulsion graph subtracted with global scale λ. The subtractive formulation here is equivalent, but λ can be set as a multiple of the baseline edge weight rather than an absolute value: λ = α × median(W_feat). This makes the parameter dimensionless and interpretable: α = 1 means full repulsion equals a typical feature edge; α = 2 means repulsion can push edges to −W_feat; etc.

Why this is better than multiplicative

  • Genuinely penalises spatial-neighbour co-assignment, not just removes reward
  • Strong enough λ forces separation regardless of indirect-path topology
  • Same τ_low / τ_high dead-zone logic applies unchanged
  • One new parameter α (or equivalently, a repulsion cap per edge)

Risks

  • Negative edges change the effective null model — K / resolution calibration needs re-checking
  • Too large α overrides the feature signal entirely; purely spatial assignment
  • Must verify leidenalg handles negative weights correctly in practice

8f. Alternative: cross-type attraction instead of same-type repulsion

A complementary framing: instead of penalising i and j for being in the same community when they are spatial neighbours, add positive edges between cells that are spatial neighbours and in different communities. This pulls the community structure toward mosaic interleaving from the positive side.

For each spatial pair (i, j) where current labels differ (c_i ≠ c_j): Add to W an attraction bonus: +β × (1 − d(i,j) / r) This rewards the partition for keeping spatial neighbours in different communities — the opposite signal from the same-type repulsion.

The challenge is that this requires the current cluster assignment to know which pairs are cross-type — introducing a second chicken-and-egg loop on top of the τ_low loop. However, a two-stage approach might work: run standard Leiden once to get a coarse partition, then add cross-type spatial attraction edges and re-run. The first pass provides the "which pairs are cross-type?" information; the second pass refines using both feature similarity and spatial interleaving reward.

This is also the argument for graph rewiring (V4). Rewiring suppressed edges to connect i to non-overlapping featurally-similar cells is functionally equivalent to strengthening cross-type spatial connections: the rewired edge connects i to a same-type cell that is spatially distant, which is exactly the pattern a complete mosaic would exhibit. V4 may be the cleanest way to implement the "cross-type spatial attraction" signal within the existing Leiden framework without explicit negative weights.

9. Open questions

What PAGA threshold θ_paga is appropriate? The PAGA weight between two same-type fragments vs. two genuinely distinct types should show a clear gap. On the toy dataset (ground truth known), we can plot the PAGA weight distribution across all pairs, coloured by same-type vs. different-type, and pick θ_paga at the valley. On real data without ground truth, a sweep over θ_paga values and inspection of which merges are accepted/rejected is needed.
Should PAGA be computed on W_feat or W_rep (the repulsion-weighted graph)? W_feat captures raw feature similarity, while W_rep has already suppressed edges between spatially overlapping cells. For the merge step, we want feature similarity — W_feat is probably the right choice. But if W_rep is used (it is the graph Leiden actually operated on), the PAGA weights will already reflect some of the spatial structure. Worth comparing both on the toy dataset.
How well does the Voronoi / convex-hull approximation reproduce real BC contact? The ~10% same-type contact C is a rough estimate from convex hulls of axonal arbors; the key point is just that it is > 0. The toy simulation uses Voronoi polygons, which by construction partition space without overlap, so within-cluster contact C may be ~0 in the simulation. If so, the iterative τ_low estimation will produce τ_low ≈ 0 for all simulated clusters — which effectively degrades to V1 with τ_low = 0. Validating the dead-zone benefit may require real data or an augmented simulation that introduces deliberate small hull overlaps.
What happens to K / resolution stability? Inserting repulsion effectively removes or downweights edges, which changes the effective resolution even at fixed res. The τ-repulsion may shift K upward (more communities). This means res needs to be re-tuned jointly with (τ_low, τ_high), complicating the parameter sweep. V4 (rewiring) is expected to be less affected since it preserves node degree — worth testing.
V4 (rewiring) vs. V2 (soft downweight): which is better for K stability? Rewiring preserves node degree and should disturb the resolution–K relationship less. Is this worth the added complexity? Compare community-size distributions and NMI against ground truth on the toy dataset.
How to validate that the spatial constraint is actually helping? Ground truth labels are available in the simulation. The primary metric should be NMI or ARI as a function of (τ_low, τ_high, n_iter). Include a null test: permute the coverage fraction values randomly before applying repulsion. Any improvement in the null test would indicate the benefit is a side-effect of changed graph topology (e.g. fewer edges → more communities) rather than the spatial signal itself.
WFAC and other cells where the spatial constraint does not apply. For very large or geometrically unusual cells (WFACs, bowtie ACs), meaningful coverage fractions between neighbouring cells of the same type may be near zero by geometry, and the iterative τ_low estimate will not be reliable. These cells should probably be flagged and excluded from the spatial repulsion term entirely. How to identify them automatically — by hull area percentile, by low within-cluster C variance, or by an explicit user-provided mask?
