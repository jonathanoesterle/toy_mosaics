MRF + Mosaic Constraints

Implementation reference — post-hoc spatial label refinement via Markov Random Fields · toy_mosaics project

Current implementation summary (as of 2026-05-27)
  • Inference: ICM (iterated conditional modes) — monotone convergence, local MAP
  • Unary term: GMM negative log-posteriors; high-confidence cells frozen before ICM
  • Pairwise term: two variants compared per config in generate_mrf_figures.py:

1. Why MRF: what Leiden and Ward missed

Both Leiden and Ward attempted to enforce the mosaic constraint during clustering — by modifying edge weights or merge costs before committing to a partition. Both failed on the anisotropic benchmark (ARI stuck at 0.931 regardless of penalty strength). The root cause was the spatial pivot problem: penalizing the nearest spatially-close wrong-type candidate does not prevent merging with any of the many other wrong-type cells that are spatially distant and therefore unpenalized.

The wrong merges happen with spatially distant cells. Any penalty that only fires for spatially close pairs cannot box out a wrong cluster that is "everywhere in feature space."
The key shift: act on the final assignment, not on the merge path Instead of penalizing individual merge decisions, operate on the complete label assignment y = (y_1, ..., y_N) and penalize configurations where two spatially adjacent cells share the same label. This is evaluated globally, after all clustering decisions are made, and can inspect the full spatial neighborhood at once.

2. The energy formulation

Treat cell type assignment as MAP inference in a pairwise MRF defined over the spatial graph. The energy to minimize is:

E(y) = Σ_i ψ_i(y_i) + λ · Σ_{(i,j) ∈ S} w_ij · φ(y_i, y_j) y_i ∈ {0, 1, ..., K-1} cell type assignment ψ_i(y_i) unary cost: how well does X_i fit cluster y_i? φ(y_i, y_j) pairwise cost: do cells i and j have conflicting types? S spatial edge set: pairs within spatial_radius w_ij ∈ (0, 1] proximity weight or coverage fraction for pair (i, j) λ global penalty strength

Lower E is better. The unary term comes from the GMM; the pairwise term encodes the mosaic prior. Finding the exact minimum is NP-hard for K ≥ 3, but ICM converges reliably for small K and sparse graphs.

3. Unary term: GMM posteriors + freezing CURRENT

The unary cost ψ_i(y_i) quantifies how well cell i's feature vector X_i fits cluster y_i:

ψ_i(k) = -log P(y_i = k | X_i) = -log [ π_k · N(X_i; μ_k, Σ_k) / Σ_{k'} π_{k'} · N(X_i; μ_{k'}, Σ_{k'}) ]

where π_k, μ_k, Σ_k are the GMM mixture weight, mean, and covariance for component k. Cells with max-posterior P(y_i = y* | X_i) ≈ 1 are unambiguous; cells with spread posteriors (max ≈ 0.4–0.6 for K=3) are candidates for spatial correction.

Confidence-based freezing

Before ICM, high-confidence cells are frozen — their labels act as anchors and are never updated:

frozen(i) iff P(y_i = y*_i | X_i) >= conf_threshold

The vast majority of cells (~95%) are frozen. This bounds the damage radius: the MRF adjusts only a small fraction of cells, so an incorrect reassignment cannot globally destabilize an already-good GMM solution.

ALTERNATIVE

Ward-based unary terms

Ward does not produce posterior probabilities naturally. Three options exist if Ward (rather than GMM) were used as the initial clusterer: distance to centroid ψ_i(k) = ‖X_i − μ_k‖²; soft Ward (fit GMM initialized at Ward centroids); or Mahalanobis distance using per-cluster covariance. The Mahalanobis option is most principled and recovers the same posterior as GMM for anisotropic Gaussian clusters.

4. Pairwise term: Potts and signed-ramp CURRENT — BOTH VARIANTS

The pairwise term encodes the mosaic constraint: two spatially adjacent cells should have different types. Both variants below are implemented and compared per config in generate_mrf_figures.py.

CURRENT

Plain Potts model

The standard choice: any same-type adjacency incurs cost proportional to proximity weight w_ij.
φ_Potts(y_i, y_j) = [y_i == y_j] Full pairwise cost: w_ij · [y_i == y_j] w_ij = proximity weight in (0, 1] (higher = closer)

This is label diversification (the opposite of the smoothing prior used in image segmentation). Every same-type adjacent pair incurs a penalty, including cells in the same mosaic tile that happen to be close — this is the source of the anti-helpful problem for cell 77 (§8).

CURRENT

Signed-ramp variant

Changes the sign of the pairwise term for low-CF same-type contacts. Intra-tile adjacency (low coverage fraction) becomes attractive; territorial overlap (high CF) remains repulsive. Addresses the Potts anti-helpful problem for cell 77.
φ_signed(y_i, y_j, CF_ij) = [y_i == y_j] · signed_ramp(CF_ij) signed_ramp(CF) = −(τ_low − CF) / τ_low for CF < τ_low (attractive: same-tile contact) = 0 for CF = τ_low = +(CF − τ_low) / (τ_high − τ_low) for τ_low < CF < τ_high (repulsive: territorial overlap) = +1 for CF ≥ τ_high Current default: tau_low = 0.30 → covers intra-tile CF range 0.20–0.26, putting those contacts in the attractive zone → invoked via MRFMosaicStrategy(signed_ramp=True, tau_low=0.30)
ALTERNATIVE

Type-asymmetric penalties

If prior knowledge about per-type same-type contact rates is available, the dead zone can be made type-specific: φ_asym(y_i, y_j, w_ij) = [y_i == y_j] · max(0, w_ij − τ_low(y_i)), where τ_low(k) is the expected CF for same-tile contacts of type k. Not implemented.

5. Inference: ICM CURRENT

ICM (Iterated Conditional Modes) is the inference algorithm. It sweeps over all unfrozen cells repeatedly, updating each to its locally optimal label:

E_local(i, k) = ψ_i(k) + λ · Σ_{j ∈ spatial_nbrs(i)} w_ij · φ(k, y_j) y_i ← argmin_k E_local(i, k) for each unfrozen cell i, each sweep

Sweep until no cell changes label (convergence). Total E(y) decreases monotonically — guaranteed to converge in finite steps to a local minimum.

Initialize

Start from GMM labels y. Freeze high-confidence cells (posterior ≥ conf_threshold).

Cell sweep

For each unfrozen cell i, compute E_local(i, k) for every label k. Assign y_i = argmin_k E_local(i, k).

Repeat until convergence

Continue sweeping until no cell changes label. Record n_iters, n_changed, violations_before / violations_after in the model dict.

ICM considers all labels for every cell at every sweep, not just cells currently in violation. This means it can spontaneously correct cells with no current same-type conflict if a neighboring change opens up a better assignment. For K=3 and N≈200 with a sparse spatial graph, convergence is typically fast (2–5 sweeps).

ICM converges to a local minimum, not the global optimum. Starting from a good GMM solution (ARI ≥ 0.93), the local minimum is usually close to the global one. For configs with worse GMM initialization, the echo-chamber effect — wrong cells reinforcing each other — can trap ICM. The monotone convergence guarantee does not protect against this.

6. Lambda and spatial radius

Lambda calibration

Lambda controls the trade-off between feature fidelity and spatial consistency. The natural scale: for a GMM unary term, the posterior gap for an ambiguous cell (K=3) is about log(3) ≈ 1.1 nats; the maximum pairwise term is w_ij · 1 ≤ 1. So λ ≈ 1 makes the penalty for a maximally-close same-type pair comparable to a 2.7× likelihood ratio.

Empirically, higher λ (e.g. λ=20) is used in the anisotropic analysis. The adaptive calibration below provides a principled alternative:

lambda_adaptive = alpha · median_posterior_gap posterior_gap(i) = P(y_i = 1st | X_i) - P(y_i = 2nd | X_i) median = over all cells (or over ambiguous cells only)
Too-large lambda: spatial coloring. If λ dominates, the algorithm tries to make every spatial neighbor pair have different labels — producing a graph coloring rather than a feature-based clustering. For K=3 in a dense mosaic, some same-type adjacent pairs are inevitable and correct. Lambda must allow this; the signed-ramp partly addresses it by making low-CF contacts attractive.

Spatial radius

The spatial graph is built from all pairs within spatial_radius. The auto-computed default:

spatial_radius = 3.0 × median(nearest-neighbour distances) → typically ~11 units for the standard configs

This is set in generate_mrf_figures.py from KDTree(dataset.centers).query(..., k=2). Increasing the radius is a possible fix for spatially isolated cells like cell 94 (§8b) but trades specificity for coverage.

7. Diagnostics: plot_mrf_diagnostics IMPLEMENTED

plot_mrf_diagnostics(dataset, result, *, ground_truth=None) returns a 6-panel figure. Each config in generate_mrf_figures.py generates one diagnostics page for the plain MRF and one for the signed-ramp variant.

The six panels cover:

  • Conflict–confidence scatter — one point per cell; x = GMM max-posterior (confidence), y = total same-type pairwise weight (conflict). Color = correct/error if ground truth provided. Four quadrants identify fixable (ambiguous + conflicted), frozen-violated, Type-B no-signal, and safe cells.
  • Spatial energy-gap map — polygons colored by E_local(i, 2nd-best) − E_local(i, best). Large gap = stable; gap near zero = about to flip.
  • Neighbourhood composition bars — for each misclassified cell, a stacked bar of neighbor type composition before and after ICM. Requires ground truth.
  • Feature-space scatter with conflict overlay — GMM decision boundaries + per-cell conflict score coloring.
  • ICM trajectory arrows — cells that changed label during ICM, shown as arrows in feature space with energy gain. Requires per-cell trajectory logging in the model dict (not yet stored).
Per-cell ICM trajectories are not yet logged. The model dict stores n_changed (aggregate) and violations_before/after, but not the per-cell label sequence. The trajectory-arrow panel is therefore currently empty or skipped. To enable it, extend MRFMosaicStrategy.fit() to append (iteration, cell_i, old_label, new_label) per flip — zero algorithmic cost.

8. Known failure modes: cells 77 and 94

After ICM, two cells remain misclassified on anisotropic.yaml (spatial_radius≈11, λ=20). Both are true type 0 but predicted type 2. They represent two distinct structural failure modes.

8a. Cell 77 — the anti-helpful pairwise problem

Cell 77 has four spatial neighbours within the radius. All four are correctly assigned. The feature model gives ~67% posterior for the wrong type (type 2).

Cell 77 spatial neighbours (after ICM): cell 123: CF=0.561 w=1.000 label=type1 (correct) cell 127: CF=0.533 w=1.000 label=type1 (correct) cell 75: CF=0.260 w=0.533 label=type0 (correct) ← same-tile territory cell 83: CF=0.202 w=0.340 label=type0 (correct) ← same-tile territory ICM energy for each candidate label (plain Potts): type2 (current wrong): unary_2 + λ · 0 = baseline (no same-type nbrs) type0 (correct): unary_0 + λ · (0.533+0.34) ≈ unary_0 + 17.5 ← PENALTY for correct assignment type1: unary_1 + λ · (1.0+1.0) ≈ unary_1 + 40 ICM chooses type2: feature model prefers it AND correct-type neighbours add penalty.
The Potts pairwise term is anti-helpful for cell 77. Cells 75 and 83 — correctly assigned type-0 neighbours — create a penalty for assigning cell 77 to type 0. From ICM's perspective, type-0 neighbours are evidence against becoming type 0. The correct interpretation — "I am in type-0 territory" — is the opposite of what the Potts model computes.

The signed-ramp variant partially addresses this: cells 75 (CF=0.26) and 83 (CF=0.20) are below or near tau_low=0.30, so their contribution to E_local(77, type0) becomes negative or near-zero instead of +17.5. Whether this is sufficient to overcome the 67% feature preference for type 2 depends on the calibration of λ and α.

8b. Cell 94 — spatial isolation

Cell 94 sits at the domain edge (y≈4.5) with zero spatial neighbours within the auto-computed radius.

Cell 94: pos=[41.05, 4.54] true=type0 pred=type2 Spatial neighbours within radius 11: none ICM: E_local(94, k) = unary[94, k] for all k → GMM label unchanged Feature posterior: ~61% for type2 (wrong) Broader neighbourhood (ground-truth labels, no CF filtering): within r=20: type0=8 type1=5 type2=6 → type0 plurality within r=30: type0=14 type1=12 type2=10 → type0 plurality within r=50: type0=43 type1=27 type2=18 → type0 clear majority

The territorial signal exists at larger radii but the spatial graph has no edges for this cell, so none of it enters the energy function. ICM is fully blind to it.

9. Ceiling analysis: what is fixable

From the empirical results on anisotropic.yaml, errors split into two categories:

Fixable (Type-A): cell 76 — now corrected

Cell 76 (GMM: type 2, true: type 0) had same-type spatial conflicts after GMM assignment: being labelled type-2 placed it adjacent to other type-2 cells. ICM detected the conflict and successfully reassigned it. This is the MRF sweet spot: ambiguous feature signal + direct spatial conflict.

Persistent errors: cells 77 and 94

Cell 77 satisfies the conflict condition after GMM assignment but the pairwise term works against correction (§8a). Cell 94 has no edges in the spatial graph and is therefore invisible to ICM (§8b). Neither is a classic Type-B (no-conflict, no-signal) error; both are structural limitations of the current formulation.

Necessary conditions for ICM correction

For ICM to correct a wrong assignment y_i = wrong, cell i must:

  1. Have at least one spatial edge — i.e., ≥1 neighbour within spatial_radius.
  2. Have a posterior gap small enough that the spatial term (λ × pairwise) can overcome the unary preference for the wrong label.
  3. Not be surrounded by other wrongly-assigned cells that would reinforce the wrong label via the pairwise term.
  4. Have the pairwise term point in the correct direction — which the Potts model does not guarantee when the cell is embedded in correct-type territory.
Maximum achievable ARI improvement from the current formulation Cell 76 was fixed (ARI improved from ~0.93 to ~0.95). Cells 77 and 94 require either the signed-ramp (for 77) or a larger radius / KDE prior (for 94) — see §10 for these extensions. The 2 remaining errors represent the practical floor for the current spatial-graph formulation at radius 11.

10. Extensions and alternatives

EXTENSION

Spatial density prior (KDE unary augmentation)

Augment the unary term with a per-type kernel density estimate computed from frozen cells. This encodes territory membership — "cells at position p are more likely to be type k if nearby high-confidence cells are predominantly type k."

ψ_aug(i, k) = ψ_GMM(i, k) − α · log [ KDE_k(centers[i]) / Σ_{k'} KDE_{k'}(centers[i]) ] KDE_k: fitted on {centers[j] : frozen[j] AND labels[j] == k} bandwidth h ≈ spatial_radius (Scott's rule over-smooths; a tight local bandwidth is needed) α: weight of spatial prior relative to feature term

Targets both failure modes: for cell 94 (no edges), the prior can act without any spatial graph edge. For cell 77, it can counteract the anti-helpful pairwise penalty by providing a positive pull toward type-0 territory.

Bandwidth is critical. With Scott's rule (~15–20 units for ~70 frozen points in 2D), the KDE over-smooths and gives nearly equal density for all types everywhere. A bandwidth close to spatial_radius (~11) is required to see territorial structure. Setting α large enough to overcome the 17.5-nat pairwise penalty at cell 77 risks over-weighting the prior near mosaic boundaries.
EXTENSION

Larger spatial radius for edge-of-domain cells

Increasing spatial_radius from ~11 to ~20 would give cell 94 approximately 19 neighbours, making it visible to ICM. The type-0 plurality within r=20 (8 vs 5+6) could then provide a corrective pull — but only if the anti-helpful problem (cell 77) is also addressed.

Tradeoff: a larger radius captures more intra-tile same-type contacts, increasing false "violations" that the dead zone must absorb. The auto-computed radius (3× median NN distance) is already principled; inflating it is a targeted patch for edge cells rather than a general improvement.

ALTERNATIVE

Loopy Belief Propagation (LBP)

LBP propagates probability distributions through the spatial graph rather than hard label assignments, producing approximate marginal posteriors P(y_i = k | X, neighbours). Each cell sends messages to neighbours:

m_{i→j}(k) ∝ Σ_{k'} exp(-ψ_i(k')) · exp(-λ · w_ij · φ(k', k)) · Π_{l ∈ nbrs(i)\{j}} m_{l→i}(k') MAP assignment after convergence: y_i = argmax_k b_i(k) where b_i(k) ∝ exp(-ψ_i(k)) · Π_{j ∈ nbrs(i)} m_{j→i}(k)

Advantages over ICM

  • Propagates spatial context beyond 1-hop (useful for long-chain configurations)
  • Produces calibrated posterior probabilities, not just MAP
  • Can escape some local minima that trap ICM

Drawbacks

  • O(N · K² · deg · iters) vs O(N · K · deg · iters) for ICM
  • Convergence not guaranteed on loopy graphs
  • Overkill for K≤6, N≤300, sparse local graphs — ICM is likely equivalent in practice

11. Open questions

Does the signed-ramp (tau_low=0.30) actually fix cell 77 on anisotropic? Cells 75 and 83 have CF=0.26 and 0.20, which are just below tau_low=0.30 — they should be in the mild-attractive zone. Whether the resulting attraction is large enough to overcome the 67% feature preference for type 2 needs to be read from the generated figures.
What is the right lambda? The posterior-gap calibration (§6) gives a principled starting point, but λ=20 is used empirically. Is there a closed-form derivation that avoids per-config tuning?
Can the KDE prior (§10) fix cell 94 without introducing new errors near mosaic boundaries? The risk is that boundary cells near a type-0 region get incorrectly pulled to type 0 even when their feature signal says otherwise. A precision/recall analysis of the KDE augmentation across all configs is needed before committing.
Is ICM sufficient, or do we need LBP? For the current configs (K≤3, N≤300, sparse local graph), ICM likely reaches the same local minimum as LBP. The question becomes relevant only if we add configs with longer spatial chains or weaker GMM initialization.
Does the KDE prior + signed ramp combination fix both remaining cells when used together? Each alone may be insufficient for cell 77 (67% feature gap is large). The combination reduces unary preference for type 2 (KDE) and converts the pairwise penalty into an attraction (signed ramp). Whether together they cross the threshold is cheap to test.
Can the MRF be used unsupervised for real data (no ground truth)? The mosaic violation rate — fraction of same-type spatial neighbour pairs above τ_high — is directly observable without labels and could serve as a surrogate objective for hyperparameter tuning (λ, tau_low).

12. Test suite design

Tests should cover three layers: smoke (shape/dtype/reproducibility), quality (ARI thresholds, violation monotonicity), and diagnostic (per-cell detail on failures).

Smoke tests

Extend the existing STRATEGIES parametrization in tests/test_clustering.py to include MRFMosaicStrategy. Verify labels shape/dtype, n_unique_labels == n_mosaics, and model dict keys: gmm, lam, n_iters, n_frozen, n_changed, violations_before, violations_after. Easy configs (example) are sufficient here.

Quality tests

For hard configs (anisotropic, elongated_clusters, high_overlap):

  • Non-regression: ARI(MRF) ≥ ARI(GMM) − 0.01. MRF must not hurt relative to its own initialization.
  • Monotone violations: violations_after ≤ violations_before. ICM is monotone by construction; this catches sign-flip bugs in λ.
  • Frozen cells unchanged: no cell with GMM max-posterior ≥ conf_threshold should change label. Guards against a frozen-mask bug.
HARD_CONFIGS = ["anisotropic", "elongated_clusters", "high_overlap"] @pytest.fixture(params=HARD_CONFIGS) def hard_dataset(request): cfg = _load_cfg(request.param) return dataset_from_config(cfg), request.param def test_mrf_ari_nonnegression(hard_dataset): dataset, name = hard_dataset gmm_result = GMMStrategy(n_clusters=dataset.n_mosaics).fit(dataset) mrf_result = MRFMosaicStrategy(n_clusters=dataset.n_mosaics).fit(dataset) ari_gmm = adjusted_rand_score(dataset.y, gmm_result.labels) ari_mrf = adjusted_rand_score(dataset.y, mrf_result.labels) print(f"\n[{name}] GMM ARI={ari_gmm:.3f} MRF ARI={ari_mrf:.3f}") assert ari_mrf >= ari_gmm - 0.01

Diagnostic fixture

An error_summary(dataset, result) helper returns one dict per misclassified cell with: cell_id, true_type, assigned_type, gmm_max_posterior, spatial_conflict_score, n_same_type_neighbours, was_frozen. Each entry is automatically interpreted as Type-A fixable / Type-A frozen / Type-B no-signal. Print via print() so it appears in pytest -s output even on pass.
