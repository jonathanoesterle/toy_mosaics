1. Problem Statement

We have a set of reconstructed retinal neurons, each described by:

  • Morphological feature vector X — measured descriptors (soma size, dendritic field diameter, stratification depth, branching statistics, etc.)
  • Spatial position — 2D location of cell center in the retinal patch
  • Cell polygon / convex hull — outline of the dendritic territory (Voronoi tile in simulation)
  • Partial ground truth labels — many cells have assigned types, but label reliability varies; a curated reliable subset can serve as anchors

The goal is to assign each cell to a cell type (cluster) such that:

  1. Cells of the same type are similar in feature space (intra-cluster compactness).
  2. Cells of the same type tile the retinal surface as a regular, non-overlapping mosaic (spatial exclusion — see Section 3).
  3. The number of recovered types is consistent with prior anatomical knowledge.

Importantly, connectivity data (synaptic partners) is available only for post-hoc validation and will not be used as input to clustering.

2. Data Summary

Retinal Ganglion Cells (RGCs)
N: ~4,000
Expected types: ~45
Avg. cells / type: ~90
Mosaic quality: High — very regular
Best-understood cell class; mosaics are among the most regular in the retina. Good benchmark for validating the approach before scaling up.
Amacrine Cells (ACs)
N: ~20,000
Expected types: >50
Avg. cells / type: ~300–400
Mosaic quality: Variable by subclass
Wide-field ACs (WFAC): very large territories → most are clipped; mosaic spacing very large → few cells per type in any patch.
Medium-field ACs (MFAC): mosaics expected for most types; some types (e.g. bowtie cells) are geometrically unusual.
Narrow-field ACs (NFAC): densely packed; clear mosaics.
Bipolar Cells (BCs)
N: ~50,000
Expected types: 15–16
Avg. cells / type: ~3,000–3,500
Mosaic quality: High — very regular, many cells per type
Largest dataset, but fewest types — the most tractable clustering problem. High cell count per type gives strong mosaic signal. Scaling is the main challenge.

3. The Mosaic Prior

The mosaic constraint is the key biological structure that distinguishes this problem from generic feature clustering. It is an exclusion constraint, not a co-location constraint:

Cells of the same type should NOT be spatial neighbors — they tile space such that each cell's territory is surrounded by cells of different types.

This is the opposite of the usual clustering assumption (similar things are nearby). Concretely:

  • The nearest spatial neighbor of any cell is typically of a different type.
  • The k-nearest same-type neighbors are regularly spaced at approximately the mosaic spacing (~1/√density).
  • Regularity can be quantified by the coefficient of variation (CV) of nearest-neighbor distances (NNDs) within a putative type — low CV = regular mosaic.
  • The Voronoi polygon area (or its equivalent circular diameter) is inversely related to local type density, and varies predictably across types of different sizes.

Implications for clustering: A standard graph where edges represent feature similarity and spatial proximity will work only if the spatial component is inverted — spatial neighbors should be repelled from the same cluster, not attracted to it. The spatial structure is a regularizer, not a grouping signal.

4. Special Challenges

Clipped cells (boundary cells) Their Voronoi / dendritic territory is truncated at the edge of the reconstructed patch. This has two effects: (a) Hull features (area, perimeter, circularity) are biased toward smaller, more irregular polygons. (b) Their spatial neighborhood is incomplete — neighbors outside the patch are missing, making the mosaic constraint unreliable for these cells. Wide-field ACs are especially affected: nearly all of them will be clipped.

Options: exclude from spatial constraint; down-weight in regularity metrics; assign type post-hoc via feature space only; treat as a separate "uncertain" category and propagate labels from confident interior cells.
Missing cells Some cells may be absent from the reconstruction due to segmentation failures or tissue artifacts. This introduces gaps in the mosaic and reduces the apparent regularity of genuine types. The NND distribution for a type with ~10% missing cells will have a heavier right tail. Approaches should be robust to O(10–20%) dropout.
Partial reconstructions / small fields If only a small patch of retina is reconstructed, rare types with large territories may have very few representatives (potentially 1–5 cells). These cannot form a statistically detectable mosaic and may not cluster reliably from features alone. They risk being absorbed into the nearest large cluster. Hierarchical approaches or low-K fallback with post-hoc splitting may be needed.
Variable mosaic quality across types Some types (e.g. bowtie ACs, some WFACs) have intrinsically irregular or geometrically unusual territories. For these, the spatial regularity prior is weak or undefined. Any algorithm that applies the mosaic constraint uniformly will either over-regularize these types or ignore genuinely irregular structure. The spatial penalty weight needs to be type-adaptive or togglable per cell.
Unreliable partial labels Many cells have existing type assignments, but label quality varies. Using noisy labels as hard constraints will propagate errors. They are most useful as soft priors or for initializing cluster centers after curation.

5. Clustering Strategies

Baseline

A — Pure Feature Clustering

Cluster cells solely based on the morphological feature vector X, ignoring spatial position entirely. Standard methods: k-means, GMM, hierarchical agglomerative clustering. Can be preceded by PCA or UMAP for dimensionality reduction / visualization.

Pros

  • Simple, fast, well-understood
  • No spatial assumptions required
  • Works out of the box with scikit-learn
  • Good baseline to compare spatial methods against

Cons

  • Ignores the most distinctive biological constraint
  • May merge types with overlapping feature distributions
  • No mechanism to distinguish e.g. two ON vs. two OFF cell types with similar morphologies
  • K must be specified; no internal guidance from mosaic structure
Best fit: BCs (few types, strong feature separation) and as a first-pass for all datasets before adding spatial constraints.
Two-stage

B — Feature Clustering + Post-hoc Mosaic Validation

Cluster in feature space first, then assess whether each resulting cluster forms a regular mosaic (low NND-CV, Poisson-consistent spacing). Iteratively split clusters with poor regularity (may contain mixed types) or merge adjacent clusters that together form a better mosaic. The spatial structure provides a quality signal, not a training signal.

Pros

  • Modular — easy to implement and reason about
  • Spatial regularity serves as interpretable validation
  • Feature clustering can be swapped independently
  • Split/merge decisions are explainable

Cons

  • Post-hoc split/merge heuristics are hard to make principled
  • No gradient signal — iterative improvement may cycle or stall
  • Regularity metrics are noisy for small clusters (few cells per type)
  • Does not jointly optimize feature and spatial objectives
Best fit: Exploratory analysis; generating hypotheses about K; validating other methods. Also useful as a final diagnostic layer on top of any clustering result.
Spatial regularization

C — Joint Feature + Spatial Penalty Objective

Add a spatial separation penalty to the clustering objective that penalizes spatially adjacent cells being assigned to the same cluster. For k-means this modifies the assignment step; for soft assignments (GMM) it adds a term to the log-likelihood. The penalty weight λ controls the trade-off. The spatial term operates via a KD-tree radius search: for each cell, neighbors within distance r incur a penalty proportional to how much responsibility they share for the same cluster.

Pros

  • Principled: spatial constraint is built into the objective
  • Can be gradient-optimized (soft version)
  • λ gives explicit control over how hard the spatial constraint is enforced
  • The constraint radius r can match the known mosaic spacing per type

Cons

  • Two hyperparameters to tune (λ, r) — and optimal values may differ by cell type
  • Sensitive to initialization; may need multiple restarts
  • Spatial penalty is isotropic — doesn't capture mosaic regularity directly, only local exclusion
  • O(N·neighbors) per iteration; slow for large N without approximations
Best fit: RGCs and NFACs where mosaic is regular and the exclusion zone radius is approximately known from dendritic field size.
Graph clustering

D — Mixed-Affinity Graph + Louvain / Leiden

Build a cell–cell affinity graph where edge weights combine feature similarity (e.g. RBF kernel on X) and spatial structure (repulsion between spatial neighbors). The spatial repulsion can be encoded as a penalty on edges between cells that are physically close — effectively negative weight or a graph constructed to link spatially distant but featurally similar cells. Run community detection (Leiden algorithm, widely used in single-cell biology via scanpy) on this graph.

Pros

  • No assumption of convex or Gaussian clusters — handles arbitrary shapes
  • Leiden is robust, parallelizable, well-tested on large single-cell datasets
  • Resolution parameter controls K — can scan resolution to find stable range
  • Graph structure naturally handles heterogeneous density across types

Cons

  • Negative edge weights are non-standard in graph clustering — need careful formulation
  • Graph construction (k, kernel bandwidth, repulsion weight) has many free parameters
  • Resolution → K mapping is non-linear and unstable near bifurcations
  • Less interpretable than model-based approaches
Best fit: ACs, where feature distributions may be non-Gaussian and types have very different scales (NFAC vs. WFAC). Naturally scales to N=20,000. Can integrate spatial repulsion via graph rewiring.
Semi-supervised

E — Anchor-seeded Clustering with Curated Labels

Use the curated reliable subset of known labels as anchors. These can be used in several ways: (1) Constrained k-means — labeled cells are forced to stay in their cluster throughout optimization. (2) Semi-supervised GMM — initialize cluster parameters from labeled cells; unlabeled cells are treated as missing data. (3) Label propagation — spread known labels to unlabeled cells via the affinity graph, with spatial repulsion encoded in graph structure. Method (2) or (3) is preferable given the noisy-label context.

Pros

  • Leverages expensive manual annotation directly
  • Dramatically better at recovering rare types with few cells
  • Anchor cells pin cluster identities — no permutation ambiguity
  • Natural place to encode literature priors (expected type sizes, feature ranges)

Cons

  • Curated labels must be truly reliable — wrong anchors propagate errors
  • Does not help for completely novel types absent from the curated set
  • Curation effort is not free; requires domain expertise per type
  • May over-fit to anchor cell morphologies if anchors are not representative
Best fit: All three datasets once a reliable curated subset is assembled. Strongly recommended as the backbone strategy — pure unsupervised is harder to trust without any anchor.
Initialization

F — Mosaic-aware Initialization

Use spatial structure to generate good initial cluster assignments, which can then be refined by any feature-based method. Key idea: seed initial cluster centers by finding spatially regular subsets — e.g. run a greedy spatial Poisson disk sampling to find candidate cell sets that are approximately regularly spaced, then check feature coherence within each candidate. Alternatively, initialize K cluster centers as the centroids of feature vectors of cells that are maximally spatially separated (a spatial max-spread initialization analogous to k-means++).

Pros

  • Good initialization rescues many clustering methods from poor local optima
  • Algorithm-agnostic: can feed into any subsequent method (GMM, Leiden, etc.)
  • Forces consistency with the mosaic prior from the start

Cons

  • Requires approximate knowledge of K or mosaic spacing per type
  • Fails for irregular types (WFACs, bowtie cells) where no regular seed can be found
  • Adds complexity to the initialization step without a clear objective
Best fit: Complement to strategies C and E. Use for RGCs and BCs where mosaic spacing is predictable.
Hierarchical

G — Coarse-to-Fine Hierarchical Clustering

Cluster at multiple scales: first separate broad functional classes (ON vs. OFF; narrow vs. wide field) using the most discriminative features, then recursively subdivide each class into subtypes. For ACs, the first split might separate NFAC / MFAC / WFAC by dendritic field diameter before attempting fine-grained typing. The mosaic constraint becomes progressively more informative at finer scales where each subtype forms its own mosaic.

Pros

  • Aligns with biological taxonomy (class → subclass → type)
  • Reduces the effective K at each level — simpler optimization per stage
  • Partial reconstructions that can't resolve fine types can still be coarsely typed
  • Coarse classes can be validated against known markers before refining

Cons

  • Early-stage splits may not align with the final type structure
  • Errors at coarse levels propagate to all fine-level assignments within that branch
  • Requires meaningful feature subsets for each hierarchical level
  • Stopping criteria (when to stop splitting) are non-trivial
Best fit: ACs — the natural size-based stratification makes coarse-to-fine splitting biologically motivated. Also useful for ACs where WFAC needs feature-only clustering while NFAC uses spatial constraints.

7. Recommended Pipeline

The strategies above are not mutually exclusive. A practical pipeline combining them:

Standardize and explore features

Z-score each feature; run PCA to check variance explained and identify redundant/noisy dimensions. Use UMAP for visual exploration of cluster structure. This is purely exploratory — UMAP output should not be used as clustering input.

Separate cell classes by dendritic field size (ACs only)

For amacrine cells, split into NFAC / MFAC / WFAC using dendritic field diameter (or hull area) as the primary discriminator. This reduces the subsequent K at each level and allows different spatial constraint strengths per class.

Curate anchor cells from existing labels

For each candidate type in the existing label set, identify a subset of high-confidence cells (interior cells, non-clipped, morphologically typical). These become the anchors for semi-supervised initialization.

Semi-supervised initialization (Strategy E + F)

Initialize cluster centers from anchor cell feature centroids. For types with no anchors, supplement with spatially-seeded initialization (Strategy F). This gives a starting point that already respects both feature structure and mosaic spacing.

Joint feature + spatial refinement (Strategy C or D)

Refine the initial assignment using either a spatial-penalty objective (Strategy C, works well for regular mosaics) or graph-based Leiden clustering (Strategy D, better for irregular types and large N). Interior cells only at this stage — clipped cells are excluded from the spatial constraint.

Post-hoc assignment of clipped and missing-neighbor cells

Assign clipped cells to their nearest cluster in feature space (no spatial constraint). Optionally allow a softer spatial weight for clipped cells that have at least some unclipped neighbors.

Mosaic quality audit (Strategy B)

For each resulting cluster: compute NND-CV, compare to Poisson expectation. Flag clusters with poor regularity as candidates for splitting. Attempt merging of small adjacent clusters with complementary spatial patterns.

Validate against connectivity (deferred)

Once connectivity data is available, check whether cells assigned to the same type share similar synaptic partner distributions. This is the ground truth — use it to identify systematic mis-assignments and iterate on the pipeline.

8. Open Questions

How to choose the spatial penalty radius r (or graph neighborhood size) per cell type? For RGCs this can be estimated from hull area; for ACs with variable field size it needs to adapt per-cell or per-cluster.
How to handle the near-total clipping of wide-field ACs? If nearly all WFACs are clipped, the mosaic constraint is unusable. Are there features (possibly derived from partially visible dendrites) reliable enough to distinguish WFAC subtypes purely from morphology?
Should types with intrinsically irregular mosaics (bowtie ACs, certain displaced ACs) be identified by a separate criterion, or should the spatial regularity prior simply be turned off for them after initial classification?
How to estimate K from data when it is not exactly known? Candidate approaches: stability of Leiden communities across resolution sweep; mosaic regularity as a function of K; silhouette score; information-theoretic criteria (BIC for GMM). These need systematic comparison on the simulation before applying to real data.
How reliable are the existing labels? Can label reliability be inferred automatically (e.g. cells far from their cluster centroid in feature space are likely mis-labelled) or does curation require per-type domain expertise?
For the simulation: how realistic are the Gaussian blob features relative to actual morphological measurements? Are the simulated feature distributions (spread, overlap, dimensionality) comparable to what we expect in real data? Should the simulation include correlated features or non-Gaussian distributions to stress-test clustering methods?
When the reconstruction covers only a small patch, types with large territories (WFACs) may have O(1–5) representatives. What is the minimum number of cells per type needed for reliable clustering? This determines the minimum usable patch size per cell class.
